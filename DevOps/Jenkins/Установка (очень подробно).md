## 1. Требования и подготовка

### Минимальные требования к серверу :
- **RAM**: минимум 2 GB (рекомендуется 4 GB для активной работы)
- **CPU**: 2 ядра (рекомендуется)
- **Дисковое пространство**: минимум 10 GB свободного места
- **ОС**: Ubuntu 22.04 LTS или 24.04 LTS

### Предварительные шаги:

```bash
# Войдите на сервер через SSH или консоль
ssh ваш_пользователь@ip_сервера

# Обновите систему
sudo apt update && sudo apt upgrade -y

# Установите базовые утилиты (если не установлены)
sudo apt install -y wget curl git software-properties-common
```

## 2. Установка Java (JDK)

Jenkins требует Java для работы. Для современных версий Jenkins LTS необходима **Java 17 или 21** .

### Установка OpenJDK 21 (рекомендуется):

```bash
# Установка Java 21 (версия с длительной поддержкой до 2029 года)
sudo apt update
sudo apt install -y fontconfig openjdk-21-jre-headless

# Проверка установки
java -version
```

**Ожидаемый вывод:**
```
openjdk version "21.0.10" 2026-01-20
OpenJDK Runtime Environment (build 21.0.10+7-Ubuntu-124.04)
OpenJDK 64-Bit Server VM (build 21.0.10+7-Ubuntu-124.04, mixed mode, sharing)
```

### Настройка переменной JAVA_HOME:

```bash
# Найти путь к Java
which java
# Пример вывода: /usr/bin/java

# Узнать фактический путь
readlink -f $(which java) | sed 's:/bin/java::'
# Вывод: /usr/lib/jvm/java-21-openjdk-amd64

# Добавить JAVA_HOME в окружение
echo "export JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64" | sudo tee -a /etc/environment
source /etc/environment

# Проверить
echo $JAVA_HOME
```

## 3. Установка Jenkins

Jenkins отсутствует в стандартных репозиториях Ubuntu, поэтому нужно добавить официальный репозиторий проекта .

### Добавление репозитория Jenkins:

```bash
# Создаем директорию для ключей (если её нет)
sudo mkdir -p /etc/apt/keyrings

# Скачиваем GPG ключ репозитория (обратите внимание на актуальный ключ 2026 года)
sudo wget -O /etc/apt/keyrings/jenkins-keyring.asc \
  https://pkg.jenkins.io/debian-stable/jenkins.io-2026.key

# Добавляем репозиторий Jenkins LTS
echo "deb [signed-by=/etc/apt/keyrings/jenkins-keyring.asc] \
  https://pkg.jenkins.io/debian-stable binary/" | sudo tee \
  /etc/apt/sources.list.d/jenkins.list > /dev/null

# Обновляем список пакетов
sudo apt update
sudo apt upgrade -y

# Устанавливаем Jenkins
sudo apt install -y jenkins
```

### Запуск и проверка Jenkins:

```bash
# Jenkins автоматически запускается после установки
# Проверим статус службы
sudo systemctl status jenkins
```

**Ожидаемый вывод (должен быть active/running):**
```
● jenkins.service - Jenkins Continuous Integration Server
     Loaded: loaded (/usr/lib/systemd/system/jenkins.service; enabled)
     Active: active (running) since Thu 2026-02-05 06:20:00 UTC; 29s ago
   Main PID: 2646 (java)
     Memory: 585.6M
```

```bash
# Проверка версии Jenkins
jenkins --version
# Вывод: 2.541.3 (или новее)

# Если служба не запустилась автоматически:
sudo systemctl start jenkins
sudo systemctl enable jenkins  # Добавляем в автозагрузку
```

> **Важно:** Jenkins использует около 600 MB RAM при запуске. Это нормально .

### Получение первоначального пароля администратора:

```bash
# Пароль нужен для первого входа в веб-интерфейс
sudo cat /var/lib/jenkins/secrets/initialAdminPassword
```
Скопируйте этот 32-значный пароль — он понадобится на шаге 5.

## 4. Настройка файрвола (UFW)

По умолчанию Jenkins работает на порту **8080**. Нужно открыть этот порт в файрволе :

```bash
# Проверяем статус UFW
sudo ufw status

# Открываем порт 8080 для Jenkins
sudo ufw allow 8080/tcp

# Убеждаемся, что SSH остаётся открытым
sudo ufw allow OpenSSH

# Если UFW не активирован, включаем его
sudo ufw enable

# Проверяем правила
sudo ufw status
```

**Ожидаемый вывод:**
```
Status: active

To                         Action      From
--                         ------      ----
8080/tcp                   ALLOW       Anywhere
OpenSSH                    ALLOW       Anywhere
```

## 5. Первоначальная настройка Jenkins через веб-интерфейс

Теперь можно приступить к настройке Jenkins через браузер.

### Шаг 1: Доступ к Jenkins

Откройте браузер и перейдите по адресу:
```
http://IP_вашего_сервера:8080
```
(Замените `IP_вашего_сервера` на реальный IP-адрес сервера)

### Шаг 2: Разблокировка Jenkins

Вы увидите экран **"Unlock Jenkins"**:

![Unlock Jenkins screen](https://via.placeholder.com/800x400?text=Unlock+Jenkins)

Введите пароль, который вы получили командой:
```bash
sudo cat /var/lib/jenkins/secrets/initialAdminPassword
```
Нажмите **Continue** .

### Шаг 3: Установка плагинов

На следующем экране **"Customize Jenkins"** выберите один из вариантов :

| Вариант | Описание | Для кого |
|---------|----------|----------|
| **Install suggested plugins** | Устанавливает рекомендуемый набор плагинов (Git, Pipeline, Blue Ocean и др.) | ✅ **Рекомендуется для новичков** |
| **Select plugins to install** | Позволяет вручную выбрать плагины | Для опытных пользователей |

Выберите **"Install suggested plugins"**. Jenkins начнёт загрузку и установку плагинов — это займёт 2-5 минут .

### Шаг 4: Создание администратора

После установки плагинов появится форма **"Create First Admin User"**:

| Поле | Что вводить |
|------|-------------|
| **Username** | Имя администратора (например, `admin`) |
| **Password** | Надёжный пароль |
| **Full name** | Полное имя (опционально) |
| **E-mail address** | Ваш email |

Заполните поля и нажмите **"Save and Finish"** .

### Шаг 5: Настройка URL Jenkins

На следующем экране **"Instance Configuration"** укажите URL, по которому будет доступен Jenkins:
- Для тестового сервера: оставьте `http://IP_сервера:8080`
- Для production: укажите доменное имя (например, `https://jenkins.ваш-домен.ru`)

Нажмите **"Save and Finish"**.

### Шаг 6: Готово!

Вы увидите страницу **"Jenkins is Ready!"**. Нажмите **"Start using Jenkins"** — вы попадёте на главную панель управления Jenkins .

## 6. Безопасность Jenkins

Базовая установка Jenkins не является безопасной для production-окружения. Обязательно выполните следующие шаги .

### 6.1 Отключение анонимного доступа

1. Перейдите в **Manage Jenkins** → **Security**
2. Убедитесь, что включена аутентификация
3. В разделе **"Authorization"** выберите:
    - **"Logged-in users can do anything"** — только авторизованные пользователи
    - Или **"Matrix-based security"** для более тонкой настройки прав

### 6.2 Ограничение доступа по сети

```bash
# Разрешить доступ к Jenkins только с определённых IP
sudo ufw allow from 192.168.1.0/24 to any port 8080
# Удалить правило для всех IP
sudo ufw delete allow 8080/tcp
```

### 6.3 Регулярное обновление

Jenkins и плагины нужно регулярно обновлять :
- Обновления Jenkins: **Manage Jenkins** → **Update Jenkins**
- Обновления плагинов: **Manage Jenkins** → **Plugins** → **Updates** → выбрать все → **Update**

### 6.4 Использование HTTPS (рекомендуется)

Для production используйте Nginx reverse proxy с Let's Encrypt (см. [Раздел 8](#8-настройка-nginx-как-reverse-proxy-опционально)).

## 7. Установка и настройка Maven

Maven нужен для сборки Java-проектов. Настроим его глобально в Jenkins.

### 7.1 Установка Maven на сервере:

```bash
# Установка Maven из репозитория Ubuntu
sudo apt update
sudo apt install -y maven

# Проверка установки
mvn -version
```

**Ожидаемый вывод:**
```
Apache Maven 3.8.7
Maven home: /usr/share/maven
Java version: 21.0.10, vendor: Ubuntu
```

### 7.2 Настройка Maven в Jenkins:

1. Перейдите в **Manage Jenkins** → **Tools** (или **Global Tool Configuration**)

2. Найдите раздел **"Maven"** → нажмите **"Add Maven"**

   | Поле | Значение |
      |------|----------|
   | **Name** | `Maven-3` (или любое понятное имя) |
   | **Install automatically** | ❌ отключите (мы установили вручную) |
   | **MAVEN_HOME** | `/usr/share/maven` |

3. Найдите раздел **"JDK"** → нажмите **"Add JDK"**

   | Поле | Значение |
      |------|----------|
   | **Name** | `JDK-21` |
   | **Install automatically** | ❌ отключите |
   | **JAVA_HOME** | `/usr/lib/jvm/java-21-openjdk-amd64` |

4. Нажмите **"Save"**

### 7.3 Настройка Maven settings (опционально — для использования Nexus/Artifactory):

```bash
# Создаём директорию для пользовательского settings.xml
sudo mkdir -p /var/lib/jenkins/.m2

# Копируем глобальный settings.xml
sudo cp /etc/maven/settings.xml /var/lib/jenkins/.m2/settings.xml

# Меняем владельца на jenkins
sudo chown -R jenkins:jenkins /var/lib/jenkins/.m2
```

## 8. Настройка Nginx как reverse proxy (опционально)

Использование Nginx перед Jenkins даёт преимущества: HTTPS, кэширование, балансировка нагрузки.

### 8.1 Установка Nginx:

```bash
sudo apt install -y nginx
```

### 8.2 Создание конфигурации для Jenkins:

```bash
sudo nano /etc/nginx/sites-available/jenkins
```

Вставьте следующую конфигурацию (замените `jenkins.example.com` на ваш домен) :

```nginx
upstream jenkins {
    keepalive 32;
    server 127.0.0.1:8080;
}

server {
    listen 80;
    server_name jenkins.example.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name jenkins.example.com;

    # Пути к SSL сертификатам (настроим позже с Let's Encrypt)
    ssl_certificate     /etc/letsencrypt/live/jenkins.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/jenkins.example.com/privkey.pem;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES256-GCM-SHA512:DHE-RSA-AES256-GCM-SHA512;
    ssl_prefer_server_ciphers off;

    client_max_body_size 100m;

    location / {
        proxy_pass         http://jenkins;
        proxy_redirect     default;

        proxy_set_header   Host              $host;
        proxy_set_header   X-Real-IP         $remote_addr;
        proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;

        proxy_http_version 1.1;
        proxy_set_header   Upgrade $http_upgrade;
        proxy_set_header   Connection "upgrade";

        proxy_connect_timeout 90s;
        proxy_send_timeout    90s;
        proxy_read_timeout    90s;
    }
}
```

### 8.3 Активация конфигурации и установка SSL:

```bash
# Активируем сайт
sudo ln -s /etc/nginx/sites-available/jenkins /etc/nginx/sites-enabled/

# Проверяем конфигурацию
sudo nginx -t

# Перезагружаем Nginx
sudo systemctl reload nginx

# Устанавливаем Certbot для SSL
sudo apt install -y certbot python3-certbot-nginx

# Получаем SSL сертификат (замените на ваш домен)
sudo certbot --nginx -d jenkins.example.com
```

### 8.4 Настройка Jenkins для работы через reverse proxy:

В Jenkins перейдите в **Manage Jenkins** → **System** → раздел **"Jenkins Location"** → установите **Jenkins URL**:
```
https://jenkins.ваш-домен.ru
```
Нажмите **Save** .

## 9. Создание первого pipeline

Создадим простой pipeline для проверки работоспособности .

### Шаг 1: Новый элемент

На главной панели нажмите **"New Item"**

### Шаг 2: Настройка элемента

| Поле | Значение |
|------|----------|
| **Enter an item name** | `hello-world` |
| **Type** | выберите **Pipeline** |

Нажмите **OK**.

### Шаг 3: Pipeline script

Прокрутите вниз до раздела **Pipeline**. В поле **"Script"** вставьте:

```groovy
pipeline {
    agent any
    
    stages {
        stage('Hello') {
            steps {
                sh 'echo "Jenkins $(jenkins --version) on $(lsb_release -ds)"'
                sh 'java -version'
                sh 'echo "Build number: ${env.BUILD_NUMBER}"'
            }
        }
        
        stage('Info') {
            steps {
                sh 'uname -a'
                sh 'free -h'
            }
        }
    }
    
    post {
        success {
            echo 'Pipeline completed successfully!'
        }
        failure {
            echo 'Pipeline failed!'
        }
    }
}
```

### Шаг 4: Запуск сборки

Нажмите **Save**, затем на странице элемента нажмите **"Build Now"**.

Через несколько секунд сборка должна завершиться успешно (зелёный кружок). Нажмите на номер сборки → **"Console Output"**, чтобы увидеть результаты .

## 10. Управление плагинами

Плагины расширяют функциональность Jenkins. Вот как управлять ими :

### Установка нового плагина:

1. **Manage Jenkins** → **Plugins** → вкладка **Available**
2. В поиске введите название плагина (например, `Docker Pipeline`)
3. Отметьте его и нажмите **"Install without restart"**

### Популярные плагины:

| Плагин | Назначение |
|--------|-----------|
| **Docker Pipeline** | Интеграция с Docker |
| **Blue Ocean** | Современный UI для Jenkins |
| **Credentials Binding** | Безопасное хранение паролей и ключей |
| **Role-based Authorization Strategy** | Ролевая модель доступа |
| **Build Timestamp** | Отображение времени сборки |
| **Email Extension** | Расширенные email-уведомления |

### Обновление плагинов:

1. **Manage Jenkins** → **Plugins** → вкладка **Updates**
2. Выберите плагины для обновления
3. Нажмите **"Download now and install after restart"**

### Удаление плагина:

1. **Manage Jenkins** → **Plugins** → вкладка **Installed**
2. Найдите ненужный плагин и нажмите **Uninstall**
3. Для полного удаления потребуется перезапуск Jenkins: `http://ваш-сервер:8080/safeRestart`

## 11. Устранение типичных проблем

### Проблема 1: Jenkins не запускается

```bash
# Проверьте логи
sudo journalctl -u jenkins -n 50

# Проверьте права на директории
sudo chown -R jenkins:jenkins /var/lib/jenkins /var/log/jenkins /var/cache/jenkins
```

### Проблема 2: Порт 8080 уже занят

```bash
# Найти процесс на порту 8080
sudo ss -tulnp | grep 8080

# Изменить порт Jenkins в /etc/default/jenkins
sudo nano /etc/default/jenkins
# Измените HTTP_PORT=8080 на HTTP_PORT=8081

# Перезапустите Jenkins
sudo systemctl restart jenkins
```

### Проблема 3: Ошибка "Unsupported Java version"

Jenkins требует Java 17 или 21:
```bash
# Проверьте версию Java
java -version

# Если версия ниже, установите Java 21
sudo apt install openjdk-21-jdk -y
```

### Проблема 4: Не удаётся установить плагины

```bash
# Проверьте подключение к интернету
ping updates.jenkins.io

# Если за корпоративным прокси, настройте прокси в Jenkins:
# Manage Jenkins → Plugins → Advanced → HTTP Proxy Configuration
```

### Проблема 5: Недостаточно памяти для сборок

Отредактируйте файл `/etc/default/jenkins`:
```bash
sudo nano /etc/default/jenkins

# Найдите и измените строку:
JAVA_ARGS="-Xmx2048m"  # Увеличьте максимальную память до 2GB

# Перезапустите Jenkins
sudo systemctl restart jenkins
```

### Проблема 6: Нет прав на запись в workspace

```bash
# Исправьте права на рабочие директории
sudo chown -R jenkins:jenkins /var/lib/jenkins/workspace
sudo chmod -R 755 /var/lib/jenkins/workspace
```
