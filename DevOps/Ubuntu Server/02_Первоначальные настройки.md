## 1. Первые шаги после установки

### Обновление системы
```bash
sudo apt update && sudo apt upgrade -y
```
Это обновит все пакеты до актуальных версий.

### Установка OpenSSH Server (если не установлен)
```bash
sudo apt install openssh-server -y
```
**Важно:** Во время установки Ubuntu Server в разделе "SSH Setup" нужно поставить галочку "Install OpenSSH server". Если вы этого не сделали — выполните команду выше.

Проверьте, что SSH сервер запущен:
```bash
sudo systemctl status ssh
```
Если не активен — запустите:
```bash
sudo systemctl start ssh
sudo systemctl enable ssh
```

## 2. Создание нового пользователя (вместо root)

Использовать root для повседневной работы небезопасно.

```bash
# Создание пользователя (замените "myuser" на ваше имя)
sudo adduser myuser

# Добавление пользователя в группу sudo (для прав администратора)
sudo usermod -aG sudo myuser
```

Теперь вы можете подключаться по SSH под новым пользователем:
```bash
ssh myuser@192.168.3.6
```

## 3. Настройка SSH для безопасности

Отредактируйте конфигурационный файл SSH:
```bash
sudo nano /etc/ssh/sshd_config
```

Внесите следующие изменения:
```
# Отключить вход под root
PermitRootLogin no

# (Опционально) Сменить стандартный порт с 22 на другой, например 2222
Port 2222
```

После изменений перезапустите SSH:
```bash
sudo systemctl restart ssh
```

## 4. Настройка UFW Firewall

Брандмауэр нужно настроить до включения, чтобы не потерять доступ.

```bash
# Разрешить SSH подключения (если сменили порт, укажите его)
sudo ufw allow 22/tcp
# Или для нестандартного порта:
# sudo ufw allow 2222/tcp

# Разрешить HTTP и HTTPS (если планируете веб-сервер)
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# Включить firewall
sudo ufw enable

# Проверить статус
sudo ufw status verbose
```

## 5. Настройка статического IP (рекомендуется)

Чтобы IP сервера не менялся после перезагрузки:

```bash
# Посмотреть имя сетевого интерфейса
ip a
# Обычно это eth0, enp0s3 или ens160

# Отредактировать конфигурацию Netplan
sudo nano /etc/netplan/00-installer-config.yaml
```

Пример конфигурации (адаптируйте под вашу сеть):
```yaml
network:
  version: 2
  ethernets:
    eth0:  # замените на ваш интерфейс
      dhcp4: no
      addresses:
        - 192.168.3.100/24  # желаемый статический IP
      routes:
        - to: default
          via: 192.168.3.1  # ваш шлюз (роутер)
      nameservers:
        addresses:
          - 8.8.8.8
          - 8.8.4.4
```

Примените настройки:
```bash
sudo netplan apply
```

## 6. Рекомендуемые утилиты

```bash
sudo apt install -y htop curl wget git build-essential
```

- **htop** — интерактивный монитор системы
- **curl/wget** — для скачивания файлов
- **git** — система контроля версий
- **build-essential** — компиляторы и инструменты сборки

## 7. Защита от взлома (Fail2Ban)

Fail2Ban блокирует IP после нескольких неудачных попыток входа:

```bash
sudo apt install fail2ban -y
sudo systemctl enable fail2ban
sudo systemctl start fail2ban
```
